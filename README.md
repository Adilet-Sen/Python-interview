## 9. Что такое глубокая и поверхностная копия? Зачем нужны? Как их сделать?

- copy.copyэто функция, которая создает поверхностную копию объекта. Неглубокая копия означает, что создается новый объект, но копируется только ссылка на объекты внутри исходного объекта, а не создаются новые копии этих объектов.

- copy.deepcopyэто функция, которая создает глубокую копию объекта. Глубокая копия создает новый объект и рекурсивно создает новые копии всех объектов внутри исходного объекта, включая любые объекты, на которые ссылаются несколько раз.

## 10. Что такое виртуальное окружение? Зачем нужны? Какие инструменты для этого используються?

- Virtualenv — это инструмент на Python, который позволяет создавать изолированные среды Python. Он позволяет устанавливать пакеты и модули Python в отдельной среде, а не в глобальной среде Python. Это полезно для создания отдельных сред для разных проектов, чтобы каждый проект мог иметь свой собственный набор пакетов и модулей, не затрагивая другие проекты.
  - venv (Python 3.3 и более поздние версии) — это встроенный модуль виртуальной среды в Python. Он позволяет создавать виртуальные среды и управлять ими непосредственно из интерпретатора Python.
  - virtualenv (Python 2 и Python 3) — это сторонний модуль виртуальной среды, предоставляющий дополнительные функции и возможности. Его можно установить с помощью pip install virtualenv.

## 11. Poetry

- Poetry — это инструмент управления зависимостями и пакетами для Python. Он разработан, чтобы помочь разработчикам управлять проектами Python, упрощая процесс установки и управления зависимостями пакетов.

С помощью Poetry вы можете определить зависимости для вашего проекта Python в одном файле с именем pyproject.toml. Затем Poetry возьмет на себя процесс установки и управления этими зависимостями для вас. Он также предоставляет команды для создания и публикации пакетов Python, а также для запуска скриптов и тестирования проектов.

- Вот некоторые из основных особенностей поэзии:

  - Управление зависимостями: Poetry позволяет вам определить зависимости для вашего проекта в одном файле, и он будет обрабатывать процесс установки и управления этими зависимостями для вас.

  - Виртуальные среды: Poetry создает и управляет виртуальными средами для вашего проекта, что позволяет вам отделять зависимости вашего проекта от глобальной среды Python.

  - Создание и публикация пакетов: Poetry предоставляет команды для создания и публикации пакетов Python, что упрощает обмен вашими проектами с другими.

  - Выполнение и тестирование скриптов: Poetry предоставляет команды для запуска скриптов и тестов, упрощая автоматизацию общих задач и обеспечивая качество вашего проекта.

## 12. Как оценивать сложность алгоритмов и почему? Что такое Big O notation?

- Оценка сложности алгоритмов важна, потому что она позволяет нам понять, как производительность алгоритма масштабируется по мере увеличения размера входных данных. Это может помочь нам выбрать наиболее эффективный алгоритм для данной задачи, а также понять ограничения алгоритма и то, как он может работать с очень большими размерами входных данных.

- Обозначение Big O — это способ выразить сложность алгоритма. Это дает нам приблизительное представление о том, сколько времени займет выполнение алгоритма, в зависимости от размера входных данных.

## 13. Какая сложность основных операций в коллекциях?

- Сложность основных операций в collectionsмодуле в Python варьируется в зависимости от конкретной операции и используемой структуры данных. Вот несколько примеров сложности общих операций в collectionsмодуле:

> deque:
>
> > append: O(1)
> >
> > pop: O(1)
> > popleft: O(1)
> >
> > rotate: O(n)
> >
> > Counter:
>
> > update: O(n)
> >
> > most_common: O(n log n)
> >
> > defaultdict:
>
> > **getitem**: O(1)
>
> OrderedDict:
>
> > **setitem**: O(1)
> >
> > **getitem**: O(1)
> >
> > popitem: O(1)

- Имейте в виду, что это всего лишь несколько примеров, а сложность других операций в collectionsмодуле может различаться. Также важно отметить, что на фактическое время выполнения операции могут влиять другие факторы, такие как размер входных данных, используемое оборудование и конкретная реализация операции.

- Чтобы определить сложность операции в collectionsмодуле, полезно рассмотреть, как операция обрабатывает входные данные и как масштабируется время выполнения операции по мере увеличения размера ввода.

## 14. Что такое self в классах?

- В Python selfэто специальная переменная, которая используется для ссылки на текущий экземпляр класса. Когда вы определяете метод в классе, первым аргументом метода всегда является self, и он используется для доступа к атрибутам и методам текущего экземпляра класса.

## 15. Зачем нужен метод super в классе?

- Методsuper используется в Python для ссылки на родительский класс класса. Он часто используется в **init**методе подкласса для вызова **init**метода родительского класса, а также для доступа к другим методам или атрибутам, определенным в родительском классе.

## 16. Как в python реализуются методы объекта, класса и статические методы? Чем они отличаються?

- Методы объекта — это методы, определенные в классе и привязанные к экземпляру класса. Они вызываются для экземпляра класса и могут обращаться к атрибутам экземпляра и другим методам, используя selfпараметр.
- Методы класса — это методы, определенные в классе и привязанные к самому классу, а не к экземпляру класса. Они вызываются в самом классе, а не в экземпляре класса, и могут обращаться к атрибутам класса и другим методам с помощью clsпараметра.
- Статические методы — это методы, определенные в классе, но не привязанные к классу или экземпляру класса. Они вызываются в самом классе, а не в экземпляре класса, и не имеют доступа к параметрам selfили cls.

## 17. В Python нет модификаторов доступа, все по умолчанию public. Как python реализуются public, private и protected методы и атрибуты?

- В Python нет явного синтаксиса для определения общедоступных, частных или защищенных методов или атрибутов. Однако можно эмулировать поведение этих типов методов и атрибутов, используя определенные соглашения об именах и использование \_символа (подчеркивания).

- Доступ к общедоступным методам и атрибутам можно получить из любого места в коде, в том числе извне класса. Обычно они определяются без какого-либо специального синтаксиса.

- Закрытые методы и атрибуты предназначены для использования только внутри класса и не предназначены для доступа извне класса. В Python приватные методы и атрибуты обычно обозначаются двойным подчеркиванием перед их именами (например, \_\_private_method). Это известно как изменение имени, и оно работает путем автоматической замены имени закрытого метода или атрибута модифицированной версией имени, доступ к которой извне класса затруднен.

- Защищенные методы и атрибуты аналогичны закрытым методам и атрибутам в том, что они не предназначены для доступа извне класса, но предназначены для доступа к подклассам класса. В Python защищенные методы и атрибуты обычно обозначаются префиксом их имен с одним символом подчеркивания (например, \_protected_method).

- Важно отметить, что эти соглашения об именах не применяются интерпретатором Python, а являются просто способом сообщить другим разработчикам, что метод или атрибут должны рассматриваться как закрытые или защищенные. Доступ к закрытым и защищенным методам и атрибутам по-прежнему возможен из-за пределов класса, но обычно это считается плохим стилем.
